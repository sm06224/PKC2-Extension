# F4. pptx-viewer（PowerPoint スライド表示）★ユーザー重点要件

**目的**: .pptx をスライド単位で表示。**F カテゴリで最も実装難度が高い**。

**UI 概要**:

- メイン: 現在スライド（16:9 アスペクト比保持）
- ナビ: Previous / Next + キーボード矢印
- サムネイルストリップ（左 or 下）
- スライド番号 / 全数
- ノートパネル（折りたたみ）
- フルスクリーンモード

**実装ノート**:

成熟した軽量 PPTX レンダラが存在しないため **カスタム OOXML パーサー** を構築。

### パース手順

1. PPTX は ZIP → **JSZip** (~100KB) で展開
2. `ppt/presentation.xml` からスライド一覧
3. 各 `ppt/slides/slideN.xml` からシェイプ・テキスト・位置
4. `ppt/slides/_rels/slideN.xml.rels` からメディア参照

### 主要 XML 要素

- `<p:sp>` (シェイプ), `<p:pic>` (画像), `<p:txBody>` (テキスト)
- `<a:off>` (位置), `<a:ext>` (サイズ) — EMU 単位 (1 inch = 914400 EMU)
- テキストラン: `<a:r>` + `<a:rPr>` (フォント、サイズ、太字、色)

### レンダリング

- SVG 推奨（スケーラブル + テキスト選択可）
- 各シェイプ → `<rect>`, `<text>`, `<image>` SVG 要素
- EMU → SVG ビューポート単位に変換

### 制約

- アニメーション、トランジション、SmartArt、チャートは非対応
- 複雑なスライドはテキストのみ表示 +「simplified view」ラベル
- **推定コード量: ~500-800 行**

**SR 依存**: SR-13, SR-14, SR-15, SR-9 | **優先度**: Tier 3

**バンドルライブラリ**: JSZip (MIT, ~100KB gzipped)
